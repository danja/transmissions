<p><b>FOAF</b> (an acronym of <b><a href="/wiki/Friend_of_a_friend" title="Friend of a friend">friend of a friend</a></b>) is a <a href="/wiki/Machine-readable_data" class="mw-redirect" title="Machine-readable data">machine-readable</a> <a href="/wiki/Ontology_(information_science)" title="Ontology (information science)">ontology</a> describing <a href="/wiki/Person" title="Person">persons</a>, their activities and their relations to other people and objects. Anyone can use FOAF to describe themselves. FOAF allows groups of people to describe <a href="/wiki/Social_networks" class="mw-redirect" title="Social networks">social networks</a> without the need for a centralised database.
</p><p>FOAF is a descriptive vocabulary expressed using the <a href="/wiki/Resource_Description_Framework" title="Resource Description Framework">Resource Description Framework</a> (RDF) and the <a href="/wiki/Web_Ontology_Language" title="Web Ontology Language">Web Ontology Language</a> (OWL). Computers may use these FOAF profiles to find, for example, all people living in Europe, or to list all people both you and a friend of yours know.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup><sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup> This is accomplished by defining relationships between people. Each profile has a unique identifier (such as the person's <a href="/wiki/E-mail_address" class="mw-redirect" title="E-mail address">e-mail addresses</a>, international <a href="/wiki/Telephone_number" title="Telephone number">telephone number</a>, <a href="/wiki/Facebook" title="Facebook">Facebook</a> account name, a <a href="/wiki/Extensible_Messaging_and_Presence_Protocol" class="mw-redirect" title="Extensible Messaging and Presence Protocol">Jabber ID</a>, or a <a href="/wiki/Uniform_Resource_Identifier" title="Uniform Resource Identifier">URI</a> of the homepage or weblog of the person), which is used when defining these relationships.
</p><p>The FOAF project, which defines and extends the vocabulary of a FOAF profile, was started in 2000 by Libby Miller and Dan Brickley. It can be considered the first <a href="/wiki/Social_Semantic_Web" title="Social Semantic Web">Social Semantic Web</a> application,<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (April 2017)">citation needed</span></a></i>&#93;</sup> in that it combines <a href="/wiki/Resource_Description_Framework" title="Resource Description Framework">RDF</a> technology with '<a href="/wiki/Social_web" title="Social web">social web</a>' concerns.<sup class="noprint Inline-Template" style="margin-left:0.1em; white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Please_clarify" title="Wikipedia:Please clarify"><span title="The text near this tag may need clarification or removal of jargon. (April 2017)">clarification needed</span></a></i>&#93;</sup>
</p><p><a href="/wiki/Tim_Berners-Lee" title="Tim Berners-Lee">Tim Berners-Lee</a>, in a 2007 essay,<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup> redefined the <a href="/wiki/Semantic_web" class="mw-redirect" title="Semantic web">semantic web</a> concept into the <a href="/wiki/Giant_Global_Graph" title="Giant Global Graph">Giant Global Graph</a> (GGG), where relationships transcend networks and documents. He considers the GGG to be on equal ground with the <a href="/wiki/Internet" title="Internet">Internet</a> and the <a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a>, stating that "I express my network in a FOAF file, and that is a start of the revolution."
</p>
<meta property="mw:PageProp/toc" />
<h2><span class="mw-headline" id="WebID">WebID</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=FOAF&amp;action=edit&amp;section=1" title="Edit section: WebID"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>FOAF is one of the key components of the <a href="/wiki/WebID" title="WebID">WebID</a> specifications, in particular for the WebID+TLS protocol, which was formerly known as FOAF+SSL.
</p>
<h2><span class="mw-headline" id="Deployment">Deployment</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=FOAF&amp;action=edit&amp;section=2" title="Edit section: Deployment"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Although it is a relatively simple use-case and standard, FOAF has had limited adoption on the web. For example, the <a href="/wiki/Live_Journal" class="mw-redirect" title="Live Journal">Live Journal</a> and <a href="/wiki/DeadJournal" class="mw-redirect" title="DeadJournal">DeadJournal</a> blogging sites support FOAF profiles for all their members,<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup> <a href="/wiki/My_Opera" title="My Opera">My Opera</a> community supported FOAF profiles for members as well as groups. FOAF support is present on <a href="/wiki/Identi.ca" title="Identi.ca">Identi.ca</a>, <a href="/wiki/FriendFeed" title="FriendFeed">FriendFeed</a>, <a href="/wiki/WordPress" title="WordPress">WordPress</a> and <a href="/wiki/TypePad" class="mw-redirect" title="TypePad">TypePad</a> services.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup>
</p><p><a href="/wiki/Yandex" title="Yandex">Yandex</a> blog search platform supports search over FOAF profile information.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">&#91;6&#93;</a></sup> Prominent client-side FOAF support was available in <a href="/wiki/Safari_(web_browser)" title="Safari (web browser)">Safari</a><sup id="cite_ref-7" class="reference"><a href="#cite_note-7">&#91;7&#93;</a></sup> web browser before RSS support was removed in Safari 6 and in the Semantic Radar<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">&#91;8&#93;</a></sup> plugin for <a href="/wiki/Firefox_(web_browser)" class="mw-redirect" title="Firefox (web browser)">Firefox</a> browser. <a href="/wiki/Semantic_MediaWiki" title="Semantic MediaWiki">Semantic MediaWiki</a>, the <a href="/wiki/Semantic_annotation" class="mw-redirect" title="Semantic annotation">semantic annotation</a> and <a href="/wiki/Linked_data" title="Linked data">linked data</a> extension of <a href="/wiki/MediaWiki" title="MediaWiki">MediaWiki</a> supports mapping properties to external ontologies, including FOAF which is enabled by default.
</p><p>There are also modules or plugins to support FOAF profiles or FOAF+SSL authorization for programming languages,<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">&#91;9&#93;</a></sup><sup id="cite_ref-10" class="reference"><a href="#cite_note-10">&#91;10&#93;</a></sup> as well as for <a href="/wiki/Content_management_systems" class="mw-redirect" title="Content management systems">content management systems</a>.<sup id="cite_ref-11" class="reference"><a href="#cite_note-11">&#91;11&#93;</a></sup>
</p>
<h2><span class="mw-headline" id="Example">Example</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=FOAF&amp;action=edit&amp;section=3" title="Edit section: Example"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The following FOAF profile (written in <a href="/wiki/Turtle_(syntax)" title="Turtle (syntax)">Turtle</a> format) states that James Wales is the name of the person described here. His e-mail address, homepage and depiction are <a href="/wiki/Web_resource" title="Web resource">web resources</a>, which means that each can be described using RDF as well. He has Wikimedia as an interest, and knows Angela Beesley (which is the name of a 'Person' resource).
</p>
